# Mobile Page Blueprints

## 1. Mobile-first intent

These blueprints define how the site should translate to phone screens without losing the editorial-poster identity.

## 2. Landing page blueprint

### Above the fold

1. compact header utility row
2. theme switcher
3. ribbon / short framing line
4. event title
5. short subtitle
6. date + location meta
7. primary CTA
8. skyline fragment

### Rules

- keep the CTA in the first viewport where possible
- do not let ornaments push the title too far down
- limit hero decorative elements to the most meaningful ones
- if the ribbon wraps, allow two lines max

## 3. Landing page section rhythm

### Recommended order

- opening premise paragraph
- logistics cards
- divider
- lore / atmosphere block
- divider
- factions / groups
- divider
- registration details / pricing / CTA repeat

### Section density

- 1 key idea per section on small screens
- avoid long uninterrupted text walls
- prefer 2-4 short paragraphs over one large block

## 4. Application page blueprint

### Header

- page title
- short reassurance copy
- optional small note about GM review and later payment link

### Form

- all fields full width
- labels above inputs
- helper text below field
- textarea heights generous
- errors immediately adjacent to fields
- submit button full width

### Sticky actions

Optional mobile sticky footer:

- primary action: submit application
- only show after form starts or after first scroll threshold
- hide when keyboard is open if it interferes

## 5. Public roster blueprint

### Structure

Each entry card:

1. display name
2. optional character name
3. optional faction name
4. optional short tag / city note

### Rules

- maintain archive-card tone
- avoid visually noisy badges
- if filters exist, collapse them into a top sheet or drawer

## 6. Payment and state pages blueprint

### Awaiting verification

- short state title
- one-paragraph explanation
- support/contact action
- optional note that verification can take a short while

### Success

- strong state title
- one-paragraph confirmation
- next steps block
- optional return to roster / home link

### Failure

- clear state title
- retry or contact instruction
- optional link back to payment message/contact

## 7. Mobile interaction rules

- no hover-dependent information
- minimum tap target 44px
- avoid nested scroll areas where possible
- keep line lengths comfortable for Hebrew
- preserve visible focus styles on keyboard/touch hybrids

## 8. Mobile typography adjustments

- hero title: allow 2-3 lines
- body text: do not drop below readable size
- metadata can be smaller but must remain clear
- tracked Hebrew labels should be used sparingly on very small screens

## 9. Mobile ornament rules

Safe on mobile:

- one skyline fragment
- a few cloud or wave marks
- subtle paper grain
- one ribbon/banner motif

Reduce or remove on mobile:

- multiple marginalia clusters
- heavy layered overlays
- wide decorative gutters
- large background illustrations behind text
